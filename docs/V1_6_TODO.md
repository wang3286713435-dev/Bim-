# v1.6 TODO - 前端数据分析与展示收口

启动日期：2026-04-26
前置版本：`v1.5.4`
当前状态：开发完成，持续热修中
版本目标：在不打乱 `v1.5` 后端主线稳定性的前提下，集中优化前端数据分析、信息分层、视觉一致性和交互体验。

## 当前进展

### 第一轮已完成

1. 机会清单 Hero 新增阶段感知概览卡，首页能直接看见：
   - 正式可跟进数量
   - 前置信号数量
   - 紧急窗口数量
   - 高完整度数量
2. 机会清单新增“当前视图”说明，明确当前筛选到底是在看：
   - 混合机会池
   - 可跟进公告
   - 前置信号
   - 变更复核
   - 归档信息
3. 机会卡片摘要已收紧：首页优先展示更短、更可判断的摘要，而不是长正文堆叠。
4. 机会卡片新增“排序依据”说明，减少用户对“为什么排在前面”的疑惑。

### 第二轮已完成

1. 数据分析页新增“业务判断摘要”，优先回答：
   - 当前有多少正式机会
   - 有多少前置信号
   - 哪个来源当前最稳
   - 哪个来源优先治理
2. 数据分析页新增“来源质量对比”，用业务口径展示来源质量分、可跟进数量、详情解析率和联系人披露率。
3. 数据分析页新增“公告阶段”分布模块，不再只看地区、预算、截止和 BIM 类型。

### 第三轮已完成

1. 代理池卡片改成更稳的响应式信息卡布局，浅色模式和窄容器下不再出现标签竖排、字段对撞和卡片溢出。
2. 代理池卡片的关键信息重新分组为“信息事实块”，弱网刷新或长文本场景下更容易读，也方便后续继续统一视觉系统。
3. 顶部版本号默认占位改成“读取中”，并提前单独回填 `/api/health`，避免刷新时短暂闪回旧版 `v1.2.0`。

### 第四轮已完成

1. 统一强化了浅色模式下的 badge / tag 对比度，补齐 `sky / rose / white` 语义映射，避免“招标公告 / 资格预审 / 已解析详情”这类关键标签发灰发虚。
2. 机会卡片和详情页头部的核心标签改成更一致的字号、字重和字距，重要状态在浅色模式下更容易一扫而过。

### 第五轮已完成

1. 筛选栏新增阶段视图预设、快捷阶段切换和分组筛选卡，把“正式公告优先 / 前置信号观察池 / 变更公告复核”直接做成可点击视图。
2. 数据分析页补齐近 7 天走势面板和预算 / 地区 / BIM 类型业务化解读，不再只停留在桶分布和总量。
3. 详情页重组为“基础信息 / 投标时间与窗口 / 联系人与投标联络 / 服务范围与资质 / 字段来源可信度”几块，并强化跟进建议与时间轴。
4. 13 寸窗口下把机会清单与右侧辅助区、详情页右侧操作区改成更保守的分栏策略，减少拥挤和换行灾难。
5. 搜索命中高亮改成轻量模式，避免摘要字段被过多高亮打碎阅读节奏。

### 第六轮热修已完成

1. 顶部状态条不再把后台队列显示成固定 `1/5` 的“伪进度”；显式刷新/扫描继续保留真实读条，后台自动队列改成轻量状态提示。
2. 扫描队列增加陈旧状态自恢复：超时未结束时自动释放前端/后端占用，并修正对应的运行记录，避免通知用户“永远在跑”。
3. 右上角通知支持点击直达项目详情；缺少详情 id 的通知会自动把关键词带回首页筛选，方便快速定位新公告。

### 第七轮热修已完成

1. 修复通知浮层在通知数量较多时无法向下滚动的问题，历史通知现在可以在面板内继续浏览。
2. 通知浮层增加最大高度约束，避免内容过多时把整个弹层撑出屏幕。

### 第八轮热修已完成

1. 首页投标机会控制区支持折叠，可在保留机会列表的同时收起概览与筛选，减少首屏占用。
2. 搜索建议与最近搜索做了去重和分工：空查询时主看最近搜索；实时输入时只展示额外建议，不再重复堆叠。
3. 首页与临时搜索页新增“清空最近搜索”按钮，便于快速清理历史词条。
4. 代理池路由口径修正：连续异常越线的出口不再显示“优先使用”，会自动降级，和真实运行策略保持一致。
5. 总公告统计文案补充“当前招采源 / 旧版历史样本”口径，减少误判为数据丢失。

### 第九轮开发中

1. 已给热点记录补充“监控源 / 历史样本”分层口径，准备彻底把旧版泛热点样本和当前 7 个招采源样本拆开。
2. `stats` 接口已补充：
   - `monitoredTotal`
   - `legacyTotal`
   - `legacyBySource`
3. 监控日志和分析页已开始使用这套口径，后续准备继续做：
   - 历史样本独立审计视图
   - 历史样本治理脚本
   - 更明确的数据范围切换

### 第十轮热修已完成

1. 定时扫描默认周期从每 `2` 小时下调为每 `8` 小时一次，减少持续长扫描对 AI 分析和详情补强的占用。
2. OpenClaw 返回混合日志时，后端会自动提取真正的 JSON 结果，不再因为输出前缀日志直接回退到规则判断。
3. 机会卡片恢复“AI 判定 / 规则回退”状态展示；即使走了 fallback，也会保留精简后的规则依据，不再看起来像 AI 模块消失。
4. OpenClaw 详情补强结果增加二次质量过滤，重点收紧单位、联系人、电话、服务范围，减少详情页抓取到正文噪声后把脏字段写入首页。

### 第十一轮热修已完成

1. 生产机 OpenClaw 配置实际只有 `main` agent，原来的 `OPENCLAW_AGENT_ID=bim-tender` 会导致所有 AI 判断直接回退到规则分析。
2. 后端现在会在配置 agent 不存在时自动回退到 `main`，避免生产机 agent 名称漂移时整轮 AI 功能失效。
3. 生产环境 `.env` 已同步修正为 `OPENCLAW_AGENT_ID=main`，后续重启或重跑重分析脚本时不会再重复踩到同一个坑。

### 第十二轮热修已完成

1. 进一步确认生产机 `main` agent 仍缺少 OpenClaw 模型认证，导致即便 agent 名称正确也会因为 `gateway token / provider auth` 问题失败。
2. 后端 AI 入口现在新增“OpenClaw 不可用时自动回退到 OpenRouter”的能力，只要 `OPENROUTER_API_KEY` 可用，就不会让 AI 判断整条链路熄火。
3. 这一层回退同样保护了详情字段提取链路：浏览器型 agent 失效时，仍可基于已抓正文走文本态结构化提取，保证详情增强不至于完全失效。

### 第十三轮热修已完成

1. 复核线上后发现 OpenClaw 偶发会返回“运行外壳 JSON”而不是正文 payload，之前会被误写成 `relevance=0 / summary=completed / reason=null` 的伪成功结果。
2. 现在后端会把“没有文本 payload 的 OpenClaw 结果”直接判为失败，不再把运行摘要误当 AI 分析结果落库。
3. AI 结果新增必填校验：缺少 `relevanceReason / summary / relevance` 时统一走失败/回退逻辑，避免再污染首页与统计。

### 第十四轮热修已完成

1. 进一步定位到 AI 成功率不稳定的主因不是模型本身，而是 OpenClaw CLI 一直复用同一条 session，导致：
   - `session file locked`
   - `live session model switch`
   - `gateway timeout`
2. 现在每一次 AI 请求都会创建独立 `session-id`，分析链路和详情补强链路不再互相争锁。
3. 这会直接提高：
   - 公告相关性判断稳定性
   - 详情页 AI 字段补强成功率

## 目标

`v1.6` 不再把重点放在新增抓取能力，而是围绕“看得懂、筛得快、判得准”来重做前端体验：

1. 让首页真正成为投标决策面板，而不是长列表堆叠。
2. 强化“正式公告 / 前置信号 / 变更补遗 / 结果归档”的视觉分层。
3. 提升数据分析页的业务解释力，而不只是展示统计数字。
4. 收掉浅色主题、窄屏布局、代理池卡片等遗留 UI bug。

## 开发边界

### 本版本主做

1. 前端页面结构与排版优化
2. 数据分析区块重做
3. 公告阶段与来源质量的可视化表达
4. 列表 / 详情 / 筛选 / Hero 的一致性收口
5. 浅色主题、移动/窄屏适配和组件级 bug 修复

### 本版本不主做

1. 不新增数据源
2. 不继续深挖 CEB 详情突破
3. 不大改后端抓取链路
4. 不做独立后台管理系统

## 验收标准

1. 首页默认视图下，用户能在 5 秒内区分：
   - 什么是正式可投标项目
   - 什么是前置信号
   - 什么是需要复核的变更公告
2. 数据分析页能回答：
   - 近期新增了多少真正可跟进机会
   - 哪些来源质量最好
   - 预算 / 地区 / BIM 类型主要集中在哪些区间
3. 浅色模式下主要页面无明显排版错位、文字对比度不足或卡片溢出。
4. 13 寸笔记本窗口宽度下，首页、详情页、分析页都能自然浏览，无明显遮挡和换行灾难。

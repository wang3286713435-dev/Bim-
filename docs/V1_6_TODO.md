# v1.6 TODO - 前端数据分析与展示收口

启动日期：2026-04-26
前置版本：`v1.5.4`
当前状态：开发完成，持续热修中
版本目标：在不打乱 `v1.5` 后端主线稳定性的前提下，集中优化前端数据分析、信息分层、视觉一致性和交互体验。

## 当前进展

### 第一轮已完成

1. 机会清单 Hero 新增阶段感知概览卡，首页能直接看见：
   - 正式可跟进数量
   - 前置信号数量
   - 紧急窗口数量
   - 高完整度数量
2. 机会清单新增“当前视图”说明，明确当前筛选到底是在看：
   - 混合机会池
   - 可跟进公告
   - 前置信号
   - 变更复核
   - 归档信息
3. 机会卡片摘要已收紧：首页优先展示更短、更可判断的摘要，而不是长正文堆叠。
4. 机会卡片新增“排序依据”说明，减少用户对“为什么排在前面”的疑惑。

### 第二轮已完成

1. 数据分析页新增“业务判断摘要”，优先回答：
   - 当前有多少正式机会
   - 有多少前置信号
   - 哪个来源当前最稳
   - 哪个来源优先治理
2. 数据分析页新增“来源质量对比”，用业务口径展示来源质量分、可跟进数量、详情解析率和联系人披露率。
3. 数据分析页新增“公告阶段”分布模块，不再只看地区、预算、截止和 BIM 类型。

### 第三轮已完成

1. 代理池卡片改成更稳的响应式信息卡布局，浅色模式和窄容器下不再出现标签竖排、字段对撞和卡片溢出。
2. 代理池卡片的关键信息重新分组为“信息事实块”，弱网刷新或长文本场景下更容易读，也方便后续继续统一视觉系统。
3. 顶部版本号默认占位改成“读取中”，并提前单独回填 `/api/health`，避免刷新时短暂闪回旧版 `v1.2.0`。

### 第四轮已完成

1. 统一强化了浅色模式下的 badge / tag 对比度，补齐 `sky / rose / white` 语义映射，避免“招标公告 / 资格预审 / 已解析详情”这类关键标签发灰发虚。
2. 机会卡片和详情页头部的核心标签改成更一致的字号、字重和字距，重要状态在浅色模式下更容易一扫而过。

### 第五轮已完成

1. 筛选栏新增阶段视图预设、快捷阶段切换和分组筛选卡，把“正式公告优先 / 前置信号观察池 / 变更公告复核”直接做成可点击视图。
2. 数据分析页补齐近 7 天走势面板和预算 / 地区 / BIM 类型业务化解读，不再只停留在桶分布和总量。
3. 详情页重组为“基础信息 / 投标时间与窗口 / 联系人与投标联络 / 服务范围与资质 / 字段来源可信度”几块，并强化跟进建议与时间轴。
4. 13 寸窗口下把机会清单与右侧辅助区、详情页右侧操作区改成更保守的分栏策略，减少拥挤和换行灾难。
5. 搜索命中高亮改成轻量模式，避免摘要字段被过多高亮打碎阅读节奏。

### 第六轮热修已完成

1. 顶部状态条不再把后台队列显示成固定 `1/5` 的“伪进度”；显式刷新/扫描继续保留真实读条，后台自动队列改成轻量状态提示。
2. 扫描队列增加陈旧状态自恢复：超时未结束时自动释放前端/后端占用，并修正对应的运行记录，避免通知用户“永远在跑”。
3. 右上角通知支持点击直达项目详情；缺少详情 id 的通知会自动把关键词带回首页筛选，方便快速定位新公告。

### 第七轮热修已完成

1. 修复通知浮层在通知数量较多时无法向下滚动的问题，历史通知现在可以在面板内继续浏览。
2. 通知浮层增加最大高度约束，避免内容过多时把整个弹层撑出屏幕。

### 第八轮热修已完成

1. 首页投标机会控制区支持折叠，可在保留机会列表的同时收起概览与筛选，减少首屏占用。
2. 搜索建议与最近搜索做了去重和分工：空查询时主看最近搜索；实时输入时只展示额外建议，不再重复堆叠。
3. 首页与临时搜索页新增“清空最近搜索”按钮，便于快速清理历史词条。
4. 代理池路由口径修正：连续异常越线的出口不再显示“优先使用”，会自动降级，和真实运行策略保持一致。
5. 总公告统计文案补充“当前招采源 / 旧版历史样本”口径，减少误判为数据丢失。

## 目标

`v1.6` 不再把重点放在新增抓取能力，而是围绕“看得懂、筛得快、判得准”来重做前端体验：

1. 让首页真正成为投标决策面板，而不是长列表堆叠。
2. 强化“正式公告 / 前置信号 / 变更补遗 / 结果归档”的视觉分层。
3. 提升数据分析页的业务解释力，而不只是展示统计数字。
4. 收掉浅色主题、窄屏布局、代理池卡片等遗留 UI bug。

## 开发边界

### 本版本主做

1. 前端页面结构与排版优化
2. 数据分析区块重做
3. 公告阶段与来源质量的可视化表达
4. 列表 / 详情 / 筛选 / Hero 的一致性收口
5. 浅色主题、移动/窄屏适配和组件级 bug 修复

### 本版本不主做

1. 不新增数据源
2. 不继续深挖 CEB 详情突破
3. 不大改后端抓取链路
4. 不做独立后台管理系统

## 验收标准

1. 首页默认视图下，用户能在 5 秒内区分：
   - 什么是正式可投标项目
   - 什么是前置信号
   - 什么是需要复核的变更公告
2. 数据分析页能回答：
   - 近期新增了多少真正可跟进机会
   - 哪些来源质量最好
   - 预算 / 地区 / BIM 类型主要集中在哪些区间
3. 浅色模式下主要页面无明显排版错位、文字对比度不足或卡片溢出。
4. 13 寸笔记本窗口宽度下，首页、详情页、分析页都能自然浏览，无明显遮挡和换行灾难。

# v1.6 TODO - 前端数据分析与展示收口

启动日期：2026-04-26
前置版本：`v1.5.4`
当前状态：待开发
版本目标：在不打乱 `v1.5` 后端主线稳定性的前提下，集中优化前端数据分析、信息分层、视觉一致性和交互体验。

## 目标

`v1.6` 不再把重点放在新增抓取能力，而是围绕“看得懂、筛得快、判得准”来重做前端体验：

1. 让首页真正成为投标决策面板，而不是长列表堆叠。
2. 强化“正式公告 / 前置信号 / 变更补遗 / 结果归档”的视觉分层。
3. 提升数据分析页的业务解释力，而不只是展示统计数字。
4. 收掉浅色主题、窄屏布局、代理池卡片等遗留 UI bug。

## 开发边界

### 本版本主做

1. 前端页面结构与排版优化
2. 数据分析区块重做
3. 公告阶段与来源质量的可视化表达
4. 列表 / 详情 / 筛选 / Hero 的一致性收口
5. 浅色主题、移动/窄屏适配和组件级 bug 修复

### 本版本不主做

1. 不新增数据源
2. 不继续深挖 CEB 详情突破
3. 不大改后端抓取链路
4. 不做独立后台管理系统

## 优先级

### P0：首页与机会清单

- [ ] 重构首页信息层级：让“投标机会清单”和“优先跟进”形成清晰主次，不再被辅助模块抢视觉中心。
- [ ] 优化 Hero 区与顶部状态条：版本号、扫描状态、运行信息更克制、更稳定。
- [ ] 收紧机会卡片摘要：首页仅保留最关键字段，避免正文过长占用列表高度。
- [ ] 强化阶段标签样式：正式公告、资格预审、前置信号、变更补遗、结果归档要一眼可分。
- [ ] 校正机会排序的解释文案，让“为什么排在前面”更明确。

### P0：数据分析页

- [ ] 重做数据分析首页，让图表围绕“投标价值判断”而不是通用热点统计。
- [ ] 增加阶段分布分析：正式公告、前置信号、变更补遗、归档信息的数量与占比。
- [ ] 增加来源质量对比：字段完整率、深抓取率、有效样本率、近 24h 稳定度。
- [ ] 增加时间趋势面板：近 7 天新机会、前置信号、已截止、优先跟进变化。
- [ ] 增加预算区间 / 地区 / BIM 类型的业务化解读，而不是只展示桶分布。

### P0：遗留 UI bug

- [ ] 修复代理池卡片在浅色模式和窄容器下的宽度溢出、字段错位问题。
- [ ] 修复部分 badge / tag 在浅色模式下对比度不足的问题。
- [ ] 修复局部组件在长文本、长标题、弱网刷新下的布局抖动。
- [ ] 检查分页区、侧栏区、详情页操作区在 13 寸笔记本窗口下的可用性。

### P1：筛选与搜索体验

- [ ] 让“公告阶段”筛选更显眼，并和“来源 / 平台 / BIM 类型 / 截止状态”形成清晰分组。

### P1：详情页

- [ ] 统一浅色 / 深色主题的语义色，而不是逐块临时修补。

### P2：后续准备

## 1.5 遗留并转入 1.6 的问题

- [ ] 代理池模块排版在浅色模式下仍不够稳。
- [ ] 首页个别模块信息密度不均匀，Hero 与列表节奏还可以更干净。
- [ ] 数据分析页目前更像运维面板，业务分析味道还不够强。
- [ ] 若干字段在长文本场景下显示仍偏拥挤，需要重新定义“摘要字段”和“详情字段”的边界。

## 验收标准

1. 首页默认视图下，用户能在 5 秒内区分：
   - 什么是正式可投标项目
   - 什么是前置信号
   - 什么是需要复核的变更公告
2. 数据分析页能回答：
   - 近期新增了多少真正可跟进机会
   - 哪些来源质量最好
   - 预算 / 地区 / BIM 类型主要集中在哪些区间
3. 浅色模式下主要页面无明显排版错位、文字对比度不足或卡片溢出。
4. 13 寸笔记本窗口宽度下，首页、详情页、分析页都能自然浏览，无明显遮挡和换行灾难。

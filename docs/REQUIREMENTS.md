# BIM 招采监控系统需求文档

## 1. 项目目标

在现有热点监控开源项目基础上，建设一个面向 BIM 招投标场景的监控系统，持续抓取广东及深圳重点招采平台的公告数据，结合 AI 识别相关性，输出可用、可筛选、可扩展的监控结果，并最终沉淀为可复用的 agent skill。

## 2. 业务目标

1. 监控 BIM 相关招采机会，而不是泛新闻热点。
2. 支持用户维护关键词，并按关键词自动巡检。
3. 系统首次启动后应具备一组默认 BIM 监控关键词。
4. 对抓取结果进行 AI 相关性识别、摘要和优先级判断。
5. 支持多来源扩展，后续可新增站点和详情页解析策略。
6. 支持将能力封装为 skill，交给其他 AI 或 agent 直接复用。

## 3. 监控范围

当前重点站点：

1. 深圳公共资源交易中心 `szggzy`
2. 深圳阳光采购平台 `szygcgpt`
3. 广东省公共资源交易平台 `guangdong`
4. 广州公共资源交易公共服务平台 `gzebpubservice`
5. 中国政府采购网 `ccgp`
6. 全国公共资源交易平台 `ggzyNational`
7. 中国招标投标公共服务平台 `cebpubservice`（v1.5 单源专项优化中）

当前关键词方向：

1. BIM 核心词：`BIM设计`、`BIM正向设计`、`BIM全过程咨询`、`BIM施工应用`、`BIM深化设计`、`BIM数字化交付`、`EPC+BIM`、`建筑信息模型`、`BIM技术服务`、`BIM咨询`、`智慧建造BIM`
2. 应用场所组合词：`机场航站楼 BIM`、`体育场馆 BIM`、`展览馆 BIM`、`医院 BIM`、`大型商业综合体 BIM`、`城市轨道交通 BIM`、`隧道 BIM`、`大型桥梁 BIM`、`城市综合管廊 BIM`、`大型污水处理厂 BIM`
3. 工程名称/业务词：`装配式建筑工程`、`工程总承包 EPC`、`总承包工程`、`全过程工程咨询`、`施工模拟`、`竣工模型交付`、`数字孪生`、`管线综合`、`智慧运维`

## 4. 核心功能需求

### 4.1 数据抓取

1. 优先使用公开接口抓取列表页。
2. 对站点请求做限频、分页、去重。
3. 对结果进行 BIM 场景过滤，排除中标结果、候选人公示、废标等无效信息。
4. 解析标题、正文、发布时间、链接、单位、预算等信息。
5. 对招采结果保存结构化字段，支持后续筛选和报表。
6. 保留后续新增来源的单一扩展入口，不把来源逻辑散落在任务代码里。

### 4.2 AI 识别

1. 支持关键词扩展。
2. 支持相关性评分。
3. 支持真假/有效性判断。
4. 支持重要度分级。
5. 支持输出摘要。

### 4.3 Firecrawl 增强

1. Firecrawl 用于详情页正文抽取。
2. 不替代列表页公开接口。
3. 仅在招采类来源上启用正文增强。
4. 对重复详情页访问做缓存，降低成本与重复请求。

### 4.4 AI Provider 路由

1. 支持 `OpenRouter` 直连。
2. 支持 `OpenClaw agent` 作为 AI provider。
3. Provider 不可用时，系统可回退为启发式打分。

### 4.5 前端展示

1. 支持来源筛选。
2. 支持关键词筛选。
3. 支持重要度和时间范围筛选。
4. 支持手动触发搜索与热点检查。
5. 支持 BIM 类型、地区、预算、截止时间、平台等招采结构化筛选。
6. 首页应优先展示“监控台”信息，而不是只给一条公告列表。
7. 首页应包含来源健康、最近扫描、结构化分布和高价值机会视图。

### 4.6 后端配置与运维

1. 支持运行时配置读取与持久化，不依赖每次改 `.env` 重启。
2. 支持来源启停配置，方便后续新增站点和临时下线不稳定来源。
3. 支持抓取运行日志、来源探测结果和运维概览接口。
4. 支持重试、熔断、冷却等后端稳态控制。

### 4.7 Skill 封装

1. 输出项目级 skill。
2. 明确核心文件、执行流程、环境变量。
3. 供其他 AI 直接调用或二次扩展。

## 5. 非功能要求

1. 结构清晰：抓取、增强、AI、入库解耦。
2. 可维护：新增站点时只增加 source adapter。
3. 可回退：AI 或 Firecrawl 失败时不阻断主流程。
4. 可验证：本地能构建、能搜索、能查看结果。
5. 可诊断：能回答“哪个来源失败了、为什么没数据、过滤掉了多少”。

## 6. 当前已知风险

1. `gzebpubservice` 现网可能被 WAF/502 拦截。
2. `guangdong` 会限频，需要控制分页速度。
3. `OpenClaw` 首次响应时间较长，需放宽 timeout。
4. 某些来源详情页可能仍需站点定制抽取策略。

## 7. 交付物

1. 可运行的前后端项目。
2. 已接入的 BIM 招采来源。
3. OpenClaw AI provider 集成。
4. Firecrawl 详情页增强。
5. 项目 skill。
6. 开发文档与需求文档。
7. 招采结构化字段与前后端筛选能力。

# Changelog

## v1.4.0-dev

开发启动：2026-04-24
目标方向：新增数据源接入能力工程化，并选择 1 个新数据源试点。

当前进展：

- 新增数据源接入 checklist、adapter 模板和诊断脚本。
- 新源默认不加入生产扫描，支持通过临时搜索/配置单源探测。
- 首批接入 `ccgp` 中国政府采购网，BIM 关键词本地验证可返回公开招标/磋商类公告。
- 首批接入 `ggzyNational` 全国公共资源交易平台实验 adapter；接口稳定但 BIM 样本当前为空，暂不建议加入生产扫描。
- 根据《招投标项目信息日报任务》接入 `cebpubservice` 中国招标投标公共服务平台列表源。
- 根据日报任务文档扩展默认筛选词：应用场所、工程总承包/全过程咨询、数字孪生、管线综合、智慧运维等。
- 中国政府采购网详情页补强：从官方详情解析预算、投标/开标截止、联系人、电话、项目编号，并写入 `ccgp-detail+rules`。
- 中国招标投标公共服务平台列表源补强：保存开标时间为截止参考，并标记 `ceb-list+rules`；详情页当前被阿里云 405 拦截，后续走浏览器/代理专项。
- 飞书同步、重算、修复脚本来源白名单扩展到 v1.4 三个全国级来源。
- 注册新源到后端来源健康、前端筛选项和飞书来源标签。
- 修复平台筛选中广州平台展示名与入库值不一致的问题，并兼容历史别名。

收口前必须完成：

- 中国政府采购网详情页字段补强。
- 中国招标投标公共服务平台 SPA 详情页字段补强。
- 新源探测失败口径细分为禁用/空结果/请求失败。
- 至少一次后端单源入库 dry-run 验证。

## v1.3.0

发布时间：2026-04-24
状态：已结束；新增数据源主线顺延到 `v1.4.0`

目标方向：后端稳定性、数据质量、广州源专项治理、AI/飞书可观测。

已完成：

- 数据治理基础：新增 `repair:tender-data` 脚本，可 dry-run / apply 修复坏链接、字段污染和历史非招采数据。
- 来源健康治理：拆分 `24h 探测失败 / 24h 轮次异常 / 最近失败原因`，前端可观测来源异常口径。
- 代理池稳定化：接入 `proxy-a`、`proxy-b`，代理池从轮询升级为健康优先 failover，坏出口失败后可自动尝试下一出口/直连。
- 广州源专项：降频、查询词分层、错误分类、低价值公告过滤、详情可信度分级、标题+正文 BIM 分类修正。
- 广州字段补强：兼容广州电子标常见截止/开标/文件获取时间写法，修复中文日期未补零导致解析失败的问题。
- AI 稳定性：OpenClaw prompt 压缩、短文本重试、并行分析、历史相关性重算并行化和规则扩词。
- AI 可观测：新增 `AiAnalysisLog`，记录 provider、状态、耗时、尝试次数、回退原因；前端 AI 面板展示成功率、回退率、平均耗时、P95 和 provider 分布。
- 详情增强：详情增强队列支持并行 worker，并在监控日志展示待处理、处理中和处理量。
- 数据分析页：新增数据质量、来源质量分布、来源治理优先级等面板。
- 飞书链路：飞书群推送和多维表同步可用，详情页新增手动推送飞书群按钮。
- 看板降噪：`ops/summary` 不再打开页面时实时探测高风险来源，改为读取最近探测和运行日志。
- OpenClaw 运维：精简 `bim-tender` agent 工作区和指令文件，归档无关历史脚本/日志/依赖。

收口保留项：

- AI 明细页暂不单独展开，只保留后端日志和汇总面板。
- 飞书完整回执时间线暂不做完整模型，保留手动推送和当前同步结果。
- 来源健康趋势图暂不扩展为长周期图表，只保留 24h 聚合和最近失败原因。
- 新增数据源不在 v1.3 主线展开，只保留 adapter/checklist 思路，正式接入放到 `v1.4.0`。

v1.4 预备方向：

- 新数据源接入 checklist 和 adapter 模板。
- 选择 1 个新数据源做试点，而不是一次性接入多个。
- 数据源接入前先完成接口探测、详情链接策略、字段模板、反爬风险和降级策略评估。

## v1.2.0

发布时间：2026-04-23
正式收口：2026-04-24
状态：已结束，最终收口标签为 `v1.2-final`

更新内容：

- 完成输入防抖搜索、命中关键词高亮，以及“仅标题搜索 / 全文字段搜索”切换
- 新增搜索建议、最近搜索记录与常用筛选视图保存
- 完成浅色主题第一轮收口，覆盖投标机会、数据分析、详情页、监控词、临时搜索主链路
- 优化投标机会清单的密度、排序、筛选与已截止显示控制
- 新增批量操作能力，支持批量标记跟进、归档、忽略
- 重做监控日志区块，补充健康信息、抓取状态、飞书启用状态与实时刷新
- 将后台扫描/刷新进度条收口到顶部 Tab 区，并压缩为轻量状态条
- 将定时抓取周期调整为每 2 小时一次
- 修复筛选下拉被 Hero 容器裁切的问题
- 修复深圳交易中心详情增强中联系人、联系电话、预算等字段缺失与脏值覆盖问题
- v1.2 收口：前端标题区版本号改为读取后端 `/api/health.version`，后端版本号从 `server/package.json` 读取，避免继续展示 `Version 1.2.0-dev`
- 详情页操作区新增“手动推送至飞书群”，用于人工把值得跟进的项目立即发送到飞书工作群
- 详情页下滑后新增圆形悬浮返回按钮，避免滚动到深处时必须回到顶部才能返回清单

## v1.5.2

发布时间：2026-04-26
状态：生产可用，CEB 详情突破进入会话复用阶段

已完成：

- 新增 CEB 详情安全会话复用机制：支持 `CEB_DETAIL_COOKIE`、`CEB_DETAIL_SESSION_FILE`、`CEB_DETAIL_USER_AGENT`、`CEB_DETAIL_REFERER`、`CEB_DETAIL_EXTRA_HEADERS`。
- CEB 详情接口默认仍关闭；开启 `CEB_DETAIL_FETCH_ENABLED=true` 后，必须提供浏览器会话 Cookie 才会请求真实详情接口，除非显式设置 `CEB_DETAIL_ALLOW_ANONYMOUS=true` 做人工探测。
- CEB 详情来源从匿名硬探测升级为 `ceb-detail-api+session`，成功后复用通用字段质量层清洗单位、预算、联系人、电话、服务范围。
- 新增 `probe:ceb-detail-session` 诊断脚本，用于验证当前 Cookie/会话文件是否可用、是否仍被 WAF、是否能返回结构化详情字段。
- 继续保留列表可信降级：会话缺失、WAF、空响应或错误时，不覆盖列表字段，不拖垮定时扫描。

## v1.5.1

开发启动：2026-04-24
生产更新：2026-04-25
状态：生产可用，继续专项优化
目标方向：中国招标投标公共服务平台专项，提升字段完整性与反爬场景下的稳定降级。

已完成：

- 启动 `cebpubservice` 单源专项，不在本版本扩展新平台。
- 验证 CEB 官方列表页稳定可抓，SPA 详情接口和 Firecrawl scrape 均会触发 WAF/JS challenge。
- CEB 列表字段补强：完整发布时间、公告类型、公告状态、服务范围、开标/截止参考。
- CEB 详情来源升级为 `ceb-list+official-table:v1.5`，前端与飞书可靠性显示“官方列表可信”。
- 新增 `probe:cebpubservice` 诊断脚本，输出样本字段覆盖和分页/反爬状态。
- 新增 `backfill:cebpubservice` 历史字段回填脚本，可 dry-run / apply 修复旧 CEB 记录。
- 修正 v1.5 新源详情增强遗漏：`ccgp`、`ggzyNational`、`cebpubservice` 已纳入原四源详情增强队列。
- 新增 OpenClaw agent 详情字段提取层，低完整度公告会把 URL 和正文交给 `bim-tender` agent 逐条核验详情页。
- Firecrawl 详情抓取增加 WAF/验证码/安全挑战页识别，避免把阻断页当成公告正文。
- 详情增强接口支持按来源补跑，并新增 `enrich:source-details` 脚本，先用于 CEB 单平台专项，后续套到另外两个新平台。
- 修复来源健康误报：关键词空结果不再归类为“疑似被拦截”，24h 失败统计只计请求失败/WAF/熔断等真实运行异常，并优先展示最近非空成功样本。
- 代理池增加主动健康探测：输出隧道可达性、出口 IP、探测延迟、最近探测错误，并在监控日志中区分“主动探测状态”和“真实抓取最近错误”。
- 代理池错误分类细化为 `502 网关异常 / 请求超时 / 隧道断开 / 连接重置 / 目标站阻断 / 认证失败`，并在路由层启用自动降级：健康代理优先，直连次之，异常代理后置，冷却中的代理最后再尝试。
- 新增 `proxyAlerts` 运维告警输出，前端监控日志可直接看到当前代理异常原因和系统的自动降级动作。
- 代理池新增连续异常阈值通知：同一代理连续命中同类异常达到阈值后，后端会输出越线告警，前端代理告警与代理池卡片同步显示“连续异常 X/Y”和触发时间。
- `ggzyNational` 增加来源级查询词分层：当 BIM / 建筑信息模型类查询在全国平台返回空结果时，自动补查 `EPC / 工程总承包 / 全过程工程咨询`，避免把“接口可达但关键词过窄”误判成来源不可用。
- 生产机执行 `cebpubservice` 历史回填：23 条旧样本统一回退到 `ceb-list+official-table:v1.5`，清理 `万元 / 0 / 张三 / 12345678901` 等历史脏值。

- `ccgp` 详情链路复核通过：生产机 6 条样本 `queued=0 / changed=0`，当前规则稳定。
- 新增通用字段质量模块 `tenderFieldQuality`，统一清洗单位、预算、联系人、电话、服务范围和详情来源可信度，作为后续复用到其他平台的基础层。
- 来源质量指标改为有效字段口径，并新增电话覆盖率、服务范围覆盖率、详情来源分布，避免脏字段被错误计入质量分。

## v1.3.0

发布时间：2026-04-24
状态：已结束；新增数据源主线顺延到 `v1.4.0`

目标方向：后端稳定性、数据质量、广州源专项治理、AI/飞书可观测。

已完成：

- 数据治理基础：新增 `repair:tender-data` 脚本，可 dry-run / apply 修复坏链接、字段污染和历史非招采数据。
- 来源健康治理：拆分 `24h 探测失败 / 24h 轮次异常 / 最近失败原因`，前端可观测来源异常口径。
- 代理池稳定化：接入 `proxy-a`、`proxy-b`，代理池从轮询升级为健康优先 failover，坏出口失败后可自动尝试下一出口/直连。
- 广州源专项：降频、查询词分层、错误分类、低价值公告过滤、详情可信度分级、标题+正文 BIM 分类修正。
- 广州字段补强：兼容广州电子标常见截止/开标/文件获取时间写法，修复中文日期未补零导致解析失败的问题。
- AI 稳定性：OpenClaw prompt 压缩、短文本重试、并行分析、历史相关性重算并行化和规则扩词。
- AI 可观测：新增 `AiAnalysisLog`，记录 provider、状态、耗时、尝试次数、回退原因；前端 AI 面板展示成功率、回退率、平均耗时、P95 和 provider 分布。
- 详情增强：详情增强队列支持并行 worker，并在监控日志展示待处理、处理中和处理量。
- 数据分析页：新增数据质量、来源质量分布、来源治理优先级等面板。
- 飞书链路：飞书群推送和多维表同步可用，详情页新增手动推送飞书群按钮。
- 看板降噪：`ops/summary` 不再打开页面时实时探测高风险来源，改为读取最近探测和运行日志。
- OpenClaw 运维：精简 `bim-tender` agent 工作区和指令文件，归档无关历史脚本/日志/依赖。

- 飞书完整回执时间线暂不做完整模型，保留手动推送和当前同步结果。
- 来源健康趋势图暂不扩展为长周期图表，只保留 24h 聚合和最近失败原因。
- 新增数据源不在 v1.3 主线展开，只保留 adapter/checklist 思路，正式接入放到 `v1.4.0`。

v1.4 预备方向：
